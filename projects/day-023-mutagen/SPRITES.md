# Mutagen Sprite Pipeline

Status: **shipped.** The plan below was executed — FLUX2-Pro PNGs now drive the primary monster rendering. SVG composition stays as a fallback for states with no matching sprite.

## What shipped

- 75 PNG sprites total in `src/assets/monsters/`:
  - 39 single-mutation sprites (every non-disabled mutation × L1, L3, L5)
  - 36 multi-mutation sprites — 2-set through 6-set, covering all three enemy archetypes (Stinger / Bulwark / Sparker) across level bands and most realistic player-build combinations
- `src/data/monsterAtlas.ts` — auto-generated by `scripts/import-monsters.mjs`, exposes `lookupAtlas(id, level)` and `lookupMulti(mutations)`
- `src/render/monster.ts` picks in order: exact multi-match → dominant single-mutation → layered SVG

## Generation (upstream)

The PNGs are produced in the sibling `artworks/` Flask server (see `artworks/README.md`). For Mutagen the relevant pieces are:

- **Prompt template** (per mutation): bold-outline 2D game-sprite style on pure-white background, 1024×1024, chibi amphibian-reptilian base body with the mutation(s) visibly expressed at the requested level (nascent / developed / apex per L1 / L3 / L5)
- **Model**: `flux2-pro` via the artworks API
- **Level descriptors**: L1 "nascent, muted, small"; L3 "developed, saturated, confident silhouette"; L5 "apex, oversized, menacing"
- **Sidecar JSON**: each `<hash>.json` carries the mutation list + seed + prompt so the atlas can be rebuilt from source of truth without re-running the API

## Import pipeline (this repo)

Run `npm run import-monsters`. The script:

1. Scans every `*.json` sidecar in the source dir (ignores the stale `index.json` cache)
2. Copies referenced PNGs into `src/assets/monsters/`, downscaled to 400×400 at quality 85 via ImageMagick (cuts bundle size ~80% vs. raw 1024²)
3. Emits a typed TS module with two `Record` indices — single-state and multi-state — plus the matchers

## Composition rules

- **Single mutations** (e.g. creation-screen preview with just Claws): show the Claws L-matched sprite directly
- **Multi-mutation monsters**: find a FLUX render of that exact mutation set if one exists (level-tolerance up to ±4 sum distance); otherwise pick the "dominant" mutation (fusion > level × cost) and show its single-mutation sprite
- **Everything else** (empty monster, mutation not covered): fall back to the original layered inline-SVG system from `src/render/sprites/index.ts`

## Coverage caveats

- `blade`, `mace`, `invisibility` are disabled in `src/data/mutations.ts` (commented out) because no FLUX sprites exist for them yet. Re-enable when sprites ship.
- Level bands L2 and L4 are not generated — the atlas falls back to the nearest available (L2 → L1 or L3, L4 → L3 or L5). Functionally invisible unless you look hard at silhouette detail.
